# `components/combat` — shared combat UI kit

**Shared combat UI kit** (UNN-492): the route-agnostic combat components rendered by BOTH the mapless encounter (`app/campaigns/[campaignShortId]/encounter/[shortId]/_components/`) and the spatial dungeon combat canvas (`app/campaigns/[campaignShortId]/dungeon/[shortId]/_components/`) — it depends on neither experience.

**Organized by concern** (folder names drop the redundant combatant-/combat- prefixes): console/ (the **headless** core — use-combat-console + the transitional generic-event dispatcher; component writes and `combat.end` use `showtime.combat.v1`), drawer/ (combatant-drawer + the actions/conditions/counters/engagement/position/vitals sections), rail/ (combatant-rail + row), conditions/ (controls = the shared ailment/battle-condition editor + state-display = its read-only peer), dialogs/ (start-combat / end-combat / end-of-turn-modal), setup/ (import-pcs-panel + panels stub), watch/ (the player-watch shared parts — owned-sheet-tabs mounts one owner-mode `EntityWriteProvider` per owned character and is rendered by **both** watches, owner-sheet-header = the identity + HP/SP + adjust-pool masthead, combat-sheet-column = both watches' combat own-sheet body, player-turn-order + watch-enemies-rail = the battlefield's turn tracker + redacted enemy roster rendered by both the mapless watch and the delve combat watch (moved from `encounter/` in UNN-604), + the owned-sheet-refresh hook that re-pulls the server-resolved sheets when the polled snapshot implies a different sheet — its pure key is `domain/combat/view/owned-sheet-refresh-key.ts`, with the rest of the snapshot folds. That key covers zone effects **and** the owned combatant's pools when the public snapshot watcher is the only available refresh path), controls/ (engagement, side-toggle, zone-enchantment), and enemies/. turn-order-strip stays at the root (exports ConsolePhase).

The watch columns compose the **shared sheet primitives** directly (`SkillCastSection`, `MechanicWidget`, `VitalsBlock`, `AdjustPoolControl`, `AffinityStrip` — all in `components/shared/` since UNN-609, so the kit imports down-tier, never into the character-sheet feature) — never a tab root. A tab root owns page chrome (its own padding, a sticky bar, viewport-keyed `lg:` card grids) that overflows a 340px rail; extract the shared body instead.

The enemies/ bestiary (UNN-346) is a three-column master-detail whose EnemyCatalogPanel is the shared inner surface: it owns **browse** only (searchable/family-filtered/level-grouped list + statblock card) and takes the third column as a `rail` node, so neither the queue's identity nor what "commit" means reaches it (UNN-541). EnemyQueueRail is the shared cart — **id-keyed** groups with `qualifier`/`detail` per row plus `headerAccessory` / `children` / `commitLabel` slots. Two consumers compose them over the shared localStorage store (domain/combat/use-staged-enemy-queue + the pure ops in domain/combat/staged-enemy-queue): the mapless browser wrapper in encounter/ (groups by enemy key, keyed by encounterId) and the delve's staging surface in dungeon/ (groups by enemy × zone, keyed by dungeonId). `EnemyCatalogDialog` (UNN-493) is the shared **mid-combat add** wrapper: a route-agnostic `Dialog` hosting the same `EnemyCatalogPanel` + `EnemyQueueRail`, staying queue-blind (the caller passes shaped rows + mutators + commit, and an optional `zonePicker` rendered in the rail's `headerAccessory`). The mapless live console consumes it — dispatching one `addParticipant` per staged copy (setups built by the pure `domain/combat/reinforcements.ts` `buildReinforcements`, which mints ids + instantiates inline entities engine-side so the app console stays engine-free) — and it's ready for the delve's deferred mid-combat add as a second consumer. Browse reads through the app-side seam (UNN-583): `domain/combat/view/enemy-catalog-view.ts` re-exports the row builder + family vocab and folds the selected key into a statblock (`selectedEnemyStatblock`), so the panel imports only `@/lib/**`. `enemy-statblock.tsx` (`EnemyStatblock`) renders an `EnemyStatblockView` (`domain/combat/view/enemy-statblock-view.ts`, projected off `resolveEntity(getEnemy(key))`); its Skills are pre-folded to `SkillCardView`s (via `domain/combat/view/skill-card-view.ts`) and rendered by the shared `ResolvedSkillRow`.

**combat conditions (ailments + battle conditions) are DM-only** — conditions/controls is the shared presentational editor; the DM drawer's conditions-section is its sole editing caller (UNN-467), the watch renders it read-only.
